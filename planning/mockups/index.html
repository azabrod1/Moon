TWO-INK PRINT · SERIF · DOUBLE-RULE FRAMES · LEADER DOTS · PLATE CAPTIONS

The full print tradition: the viewport is an engraved plate with a double-rule frame and a live caption ("PLATE VII — THE EARTH from Mare Imbrium…"). Judge: gorgeous artifact or too literary for an interactive app? Note the primes: ∅ 1°52′.

DPhosphor Ephemeris

OPEN ↗
AMBER PHOSPHOR MONO · SCANLINES · BLINKING CURSOR · WIREFRAME EARTH

One typeface, one color, maximum instrument honesty — the ephemeris as a living terminal. Earth renders as a phosphor wireframe; the panel is a character grid; SCAN> shows the real chunked event search as a progress bar. Deliberately breaks the no-glow rule to test it.

Judge: the nerd ceiling — does it thrill, or does it read as retro cosplay next to the photoreal sky?

EDark-Site

OPEN ↗
RED-LIGHT NIGHT MODE · NO BOXES · UI DIMMER THAN THE BRIGHTEST STAR

Designed like astronomers work: shown with red light ON (note the toggle, top right). Panels aren't boxes — just aligned text and hairlines floating on the sky, the chrome never outshining a star. The sky stays full-brightness; everything else whispers.

Judge: as the *default* it's radical — but even if not picked, do you want this as a mode? (Its brightness discipline should survive regardless.)

RECOMMENDATION

Pick A as the base. It's the only one that holds the full range the app actually spans — cinematic 3D, dense ephemeris data, and narrative event hints — without forcing one register everywhere. Then steal: B's segmented rate control (the clearest speed UI of the five), C's plate captions as the export feature and its primes typography (1°52′), D's scan-progress line for the chunked event search, and E's brightness discipline + red-light mode as a settings toggle. Strictly, the contest is A vs B vs C — D exists to stress-test the no-glow rule and E auditions a feature mode; seeing them rendered settles those questions too. B, C, D as full themes each over-commit to one register; A leaves room for the others as moments.
